Use plain JEDEC-compatible desktop UDIMM memory. Avoid SO-DIMM, ECC,
registered/buffered memory, DDR5, and XMP-only kits.

** Expansion

#+begin_src text

J3701: M.2 WLAN/BT PCIe x1, in use by Intel AX200
U3700: M.2 PCIe x1, available
#+end_src

The GPU slot exists, but this is still an OEM Lenovo board with a 180W PSU. Any
GPU should be low-profile, short, slot-powered, and should not require 6-pin or
8-pin PCIe power. RAM is the highest-value upgrade for this machine.

* Ownership

This repo owns the human-edited configuration. Generated state stays outside the
repo and should not be edited directly.

** System layer

File:

#+begin_src text
guix/systems/guixpc.scm
#+end_src

Owns:

- kernel and boot arguments
- bootloader
- filesystems
- users and groups
- system packages
- system services such as SSH, Bluetooth, Xorg, elogind

Apply with:

#+begin_src sh
gsr
#+end_src

** Home layer

File:

#+begin_src text
guix/home/guixpc.scm
#+end_src

Owns:

- user-level Guix Home services
- user-level service profile
- session services such as DBus, PipeWire, WirePlumber, PipeWire-Pulse
- live symlinks from selected dotfiles back into this repo
- Fish interactive configuration and native prompt

Apply with:

#+begin_src sh
ghr
#+end_src

Current live pointer:

#+begin_src text
~/.config/guix/home.scm -> ~/Projects/guix-dotfiles/guix/home/guixpc.scm
~/.guix-home             -> generated Guix Home profile in /gnu/store
#+end_src

Do not edit =~/.guix-home= directly.

** Application config

Files:

#+begin_src text
emacs/
kitty/
ranger/
gdb/
guile/
shell/
x11/
assets/
#+end_src

Owns normal dotfile behavior. Most changes here apply by reloading the affected
program. A Guix rebuild is only needed when a package or service declaration
changes.

Guix Home recreates the live app symlinks to these files during =ghr=. It does
not replace them with store snapshots, so normal repo edits remain live.

Fish is configured through Guix Home because it is the account login shell. The
prompt is native Fish, not Starship. Fastfetch is used as an interactive shell
greeter, so it appears in kitty or any other terminal that starts an interactive
Fish shell.

* Apply

Use the smallest command that matches the changed layer.

System changes:

#+begin_src sh
gsr
#+end_src

Home service changes:

#+begin_src sh
ghr
#+end_src

Both layers changed:

#+begin_src sh
gsr
ghr
#+end_src

Regular application config changes usually need only a program reload.

* Live symlinks

The important live files are symlinks back into this repo:

#+begin_src text
~/.config/guix/channels.scm       -> guix/channels.scm
~/.config/guix/home.scm           -> guix/home/guixpc.scm
~/.config/guix/systems/guixpc.scm -> guix/systems/guixpc.scm
~/.emacs.d/init.el                -> emacs/init.el
~/.emacs.d/exwm.el                -> emacs/exwm.el
#+end_src

The app and shell symlinks are managed by =guix/home/guixpc.scm=. The Guix
entrypoint symlinks under =~/.config/guix= are bootstrap links and intentionally
stay outside the Guix Home activation service so =gsr= and =ghr= always read the
editable repo files.
