* Outcome

P3.2 implements the offline half of the visual-token contract without changing
the running desktop.  Inert Scheme data are read by a pure Scheme module,
validated against an exact schema, and rendered deterministically into six
target formats:

1. an Emacs adapter;
2. one complete Kitty configuration;
3. a Fish syntax, pager, and prompt fragment;
4. GTK 3 settings;
5. GTK 4 settings; and
6. same-shell X-session/compositor/wallpaper data.

The synthetic fixture remains deliberately unmistakable: every fixture output
begins with =SYNTHETIC FIXTURE - DO NOT INSTALL=.  Fixture-only checkpoint
=3627f0b9c98e01279e8aafa42ef81128686f3ade= is pushed, clean, and equal to
=origin/main= at ahead/behind =0/0=.

Rafael then accepted the final production choice, 11 pt UI text.  Published
checkpoint =b74a98d736b6dfbca421b115d1a0c979d707a853= evolves the contract
to schema version 2, adds =theme/tokens.scm=, pins exact Modus source
provenance, freezes every production semantic role and ANSI slot, and adds
deterministic production goldens and negative controls.  It was verified clean
and equal to =origin/main= at ahead/behind =0/0=.  Production rendering is
allowed only for the exact accepted identity; changing a provenance field,
role, ANSI value, or accepted policy value is a validation error.

Neither source checkpoint wires a generated file into Guix Home.  No package
change, Home candidate build, generation, activation, process restart, GUI
launch, or reboot is evidence of P3.2.

* Source architecture

** Canonical data and reader

=fixtures/theme/valid/tokens.scm= and =theme/tokens.scm= each contain one
quoted association list.  The first is synthetic test data; the second is the
accepted production datum.  The reader in =guix/modules/sk/theme.scm= accepts
exactly one inert quoted datum and EOF.  It never =primitive-load=s or
evaluates token contents.

Schema version 2 covers provenance, semantic roles, ANSI colors, typography,
desktop policy, per-target calibrations, repository-relative assets, contrast
policy, structured Fish mappings, and the exact ordered target set.  Missing,
unknown, recursively duplicated, shared, and cyclic data are rejected.

** Reviewed terminal contrast exception

Semantic UI roles retain the accepted 7:1, 4.5:1, and 3:1 obligations.  Fourteen
of the sixteen ANSI foregrounds must also clear 3:1 against the production
canvas.  The exact Modus terminal =black= and =bright-black= endpoints are the
only exceptions: their ratios against =#0d0e1c= are approximately 1.096696
and 2.733641 respectively.  They are terminal dark/dim endpoints, not semantic
UI text or component roles.  Their exact values remain frozen, so the exception
cannot silently expand to another ANSI slot or semantic pair.

** Pure module boundary

Importing =(sk theme)= reads no live state and writes no file.  Its public
surface returns validation records, contrast ratios, or deterministic strings:

- =sk:read-theme=;
- =sk:theme-validation-errors=;
- =sk:theme-asset-errors=;
- =sk:theme-contrast-ratio=;
- =sk:render-theme=; and
- =sk:render-all=.

Asset existence is the one explicit read-only filesystem boundary.  Callers
must supply an asset root to =sk:theme-asset-errors=.  Production rendering is
permitted only after schema, accepted-identity, contrast, and asset validation;
rendering still performs no filesystem write or live-state lookup.

** Target boundaries

- Emacs loads =modus-vivendi-tinted= first, supplies a frozen adapter palette,
  and owns fixed/UI face families without starting EXWM.
- Kitty is a complete file: it has no include form, exactly one
  =allow_remote_control no= directive, and retains the accepted security and
  nonvisual base.
- Fish emits every selected syntax, pager, and prompt variable from structured
  semantic role/style records.
- GTK 3 uses =gtk-application-prefer-dark-theme=true=.  GTK 4 uses the
  =gtk-interface-color-scheme=dark= property introduced in GTK 4.20 rather
  than copying GTK 3's deprecated mechanism.
- X output is explicitly same-shell data.  It starts no process, exports only
  the cursor variables that are meant to reach child applications, and does
  not claim general environment ownership.
- Qt is absent from the schema, renderers, fixtures, packages, and variables.
