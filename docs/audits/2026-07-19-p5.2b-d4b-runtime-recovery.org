The corrected loader checker wraps only =load-exact-loader-prefix= in:

#+begin_src scheme
(dynamic-wind
  gc-disable
  (lambda ()
    (load-exact-loader-prefix %fused-program))
  gc-enable)
#+end_src

fails.  Normal GC must be restored before any source packet, production
suite, or canary is evaluated; the correction is not a whole-process GC
suppression.

loading the store program.  Its =dynamic-wind= after thunk restores that
caller module first and calls =gc-enable= second.  This preserves both module
context and normal collection across the expected prefix refusal.
