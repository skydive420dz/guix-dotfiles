P2.M1a is *implemented and fixture-verified at the source checkpoint*.  It is
not runtime-verified and is not complete.  This record owns only repository
implementation, synthetic fixtures, and source gates.  It does not claim that

* Scope delivered

The checkpoint adds:

The command reuses the existing UDisks service, system-profile =busctl= and

timer, generation, or mutable state.

* Fixed read surface

The CLI accepts only =--json=, =--summary=, =--allow-activate=,
=--cached-only=, and =--help=.  It accepts no device, object path, D-Bus
service, interface, property, method, or executable argument.

Every observation uses this source-fixed sequence:

1. one =org.freedesktop.DBus.Properties.Get= call resolves the Drive object
   privately from the fixed block object;
2. eleven =Properties.Get= calls read the reviewed =Drive.Ata= properties; and
3. one =SmartGetAttributes= call reads its cached tuple array with an empty
   options dictionary.

=--summary= retains all thirteen cached reads so its classification is
identical to the full view; it only omits selected-attribute rendering.
=--cached-only= applies =--auto-start=no= to every child.  Ordinary mode
applies =--auto-start=yes= explicitly.  Both prohibit interactive D-Bus
authorization and bound the request to five seconds.

The production source contains no SMART refresh, SMART enable/disable, SMART
self-test, power-management, erase, generic property, generic method,
object-manager, introspection, broad-dump, shell-command, or user-supplied
device path.

* Process and failure boundary

Guile launches the fixed =/run/current-system/profile/bin/busctl= executable
directly with an argument vector.  It never constructs a shell command.
Children receive =/dev/null= for stdin and stderr; stdout is captured in
memory only.  Property responses are bounded to 4 KiB and the cached attribute
response to 128 KiB.  A seven-second monotonic parent deadline surrounds the
child, including the period after pipe EOF, and every exceptional exit path
kills and reaps an unreaped child.

The short-lived private Drive-path argument remains observable to a
sufficiently privileged concurrent process.  The implementation therefore
does not claim to hide that transient argv boundary.  It does prevent the path
and raw child output from entering shell history, terminal output, retained
logs, renderer records, or exception messages.

Child exit, exec failure, signal, timeout, oversize output, invalid UTF-8,
malformed JSON, wrong signatures/types/arities, and privacy rejection map to
fixed identifier-free statuses.  A normal child failure under
=--cached-only= means only that no usable cached source was obtained; it does
not assert why the source was unavailable.

The command constructs a complete sanitized result before performing its only
output write.  An output-port failure is not retried as a second render.

* Interpretation and output contract

The default renderer is human-readable.  =--json= emits schema
=sk-guix-disk-health/v1=.  Output is closed to reviewed SMART values, ten
selected cached attributes, explicit units, cache age/state, activation
policy, view, protocol version, and one classification:

- =favorable=: no selected failing indicator is present in a fresh,
  interpretable snapshot;
- =unknown=: cache age/state or one selected value prevents that conclusion;
  or
- =failing-indicator=: at least one reviewed current or historical indicator
  is nonzero/failing.

These are point-in-time reader classifications, not whole-drive diagnoses.
All valid observations exit zero regardless of classification so future
alerting policy remains a separate slice.  UDisks self-test =success= is
rendered =success-or-never= and never claims that a test ran.

The ten selected attribute names and units are mandatory and intentionally
GuixPC-specific.  Their absence or unit drift is a protocol failure, not an
unknown generic-drive result.  Reusing the command for another drive therefore
requires a new fixture and explicit attribute-contract review.

Operational exits are:

| Code | Meaning |
|------+---------|
| 0 | valid observation rendered |
| 64 | CLI usage error |
| 65 | protocol, output-bound, UTF-8, or privacy failure |
| 69 | no usable cached source, including =SmartUpdated= zero |
| 70 | internal invariant failure |
| 74 | exec, signal, timeout, or transport failure |

The cache becomes =stale/unknown= after 1,200 seconds.  A future timestamp is
=future/unknown=.  Zero/negative sentinels retain their documented unknown
meaning instead of being rewritten as success or failure.

* Offline evidence

The focused gate passes 185 checks.  Its UDisks protocol cases use a fully
synthetic fixture and injected fake runner; they cannot invoke a subprocess.
They prove:

- exact thirteen-call order, argument vector, output bounds, and both
  activation policies;
- only =Properties.Get= and cached =SmartGetAttributes= are reachable;
- human, JSON, full, summary, cached-only, allow-activate, help, and usage
  contracts;
- fresh, exact stale-boundary, stale, future, unknown, favorable, and
  failing-indicator classifications;
- unavailable cache and fixed runner-kind/error mappings;
- malformed JSON/trailing document, missing/extra/duplicate keys, signatures,
  scalar types, array/tuple arity, ranges, selected units, duplicates,
  omissions, and future expansion rejection; and
- no private Drive path, MAC-shaped value, UUID-shaped value, or identifier
  key reaches either renderer.

Separate transport probes execute the known Guile test executable only.  They
verify clean success, nonzero exit with both output streams discarded,
pre-EOF timeout, post-EOF timeout, oversize output, invalid UTF-8, signal
termination, and missing-executable handling.  They cannot resolve or execute
=busctl=.

All retained fixture values are synthetic.  They contain no real model,
serial, WWN, UUID, PARTUUID, by-id path, MAC address, or host identity.
