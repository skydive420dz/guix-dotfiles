* Scope decision

The accepted D-34 floor is current, booted System when different, five newest
additional distinct closures, and every explicit pin.  The older =plan=
reporter lists only 44 same-target duplicate links; it is not a P5.2 command
source.  The estimator lists all 108 links outside the accepted floor.

This review divides those candidates by profile:

1. P5.2a selects all 30 Home candidates and then stops for full verification.
2. P5.2b holds all 78 System candidates until its deletion and bootloader
   boundary is designed and fixture-tested separately.
3. Pull has no candidate.

This division makes the first manual cycle unprivileged and link-only, gives
the postflight a small exact state transition, and prevents an unrelated
bootloader rewrite from hiding inside generation maintenance.

* System-command defect discovered during preflight

The installed Guix =system delete-generations= action is not link-only.  At
revision =a8391f2=, =(guix scripts system)= calls
=delete-matching-generations= first and then =reinstall-bootloader=.  The
latter lowers and builds a boot-configuration derivation and installs its
configuration.  Failure can therefore occur after links have already been
deleted.

More importantly, =reinstall-bootloader= reconstructs only a default
=bootloader-configuration= around the detected bootloader.  This machine's
declared configuration is not default:

- =grub-efi-bootloader= targets =/boot/efi=;
- its theme resolution is =3440x1440=;
- its ordered gfx modes are =3440x1440=, =2560x1440=, =1920x1080=, and
  =auto=; and
- it carries the accepted =us= keyboard layout with =caps:escape=.

The live =/boot/grub/grub.cfg= contains the corresponding ultrawide gfx mode
and generated keymap.  Its reviewed SHA256 is:

#+begin_example
6e7fe604ee58723c7e321c3a07a645a2017cfcf49bac62b3ecb0c0559ca30671
#+end_example

The live =/var/guix/gcroots/bootcfg= target is:

#+begin_example
/gnu/store/hdwc6z0wdhgqimxqg1gzkznp9irbkwhl-grub.cfg
#+end_example

Running the official System command would exceed P5.2's reviewed link-only
boundary and could silently discard declared boot presentation/input settings.
Using =--dry-run= is not a safe preview: the installed delete-action branches
do not gate generation deletion on that option.  Raw System unlinking is also
not promoted here because it needs a fixture-tested exact-tuple guard and a
decision about temporarily stale GRUB menu entries.

P5.2b must therefore choose one of two separately reviewed designs:

1. a narrow exact-link executor that proves every candidate tuple, retains
   every recovery tuple, and leaves GRUB plus =bootcfg= byte-identical while
   recording stale non-retained menu entries; or
2. a configuration-aware bootloader regeneration path that consumes the real
   repository configuration rather than Guix's default reconstruction.
