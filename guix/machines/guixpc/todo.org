Accepted baseline:

- Xorg starts on =HDMI-A-0= using the explicit =3440x1440_100= modeline.
- =xrandr= is fallback-only, not the normal path.
- =shell/xinitrc= starts wallpaper and Picom before fullscreen Emacs/EXWM.
- Emacs uses =-fs= and loads =~/.exwm= directly instead of using the Guix =exwm=
  wrapper's hardcoded =-mm= startup.
- Picom is scoped to Emacs opacity: =picom --backend glx --vsync --opacity-rule
  '85:class_g = "Emacs"'.

Rejected experiment:

- Do not add =amdgpu= to =initrd-modules= on this machine. It pulled AMDGPU/DRM
  into the initrd but made X fall back to a generic =default= 1024x768 screen.
  Reverting returned the accepted 3440x1440 state.

Remaining cosmetic issue:

- GRUB and earliest boot text are still not ultrawide. Diagnose manually with
  GRUB =videoinfo= before changing bootloader generation. If 3440x1440 is not
  listed there, Guix =gfxmode= cannot force it.
